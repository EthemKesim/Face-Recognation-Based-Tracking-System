# Local Attendance Admin Dashboard

This web app is a separate local admin layer for the existing Python face recognition project. It reuses the original SQLite database and log file, while adding a protected login for the dashboard and a web-based employee delete action.

## What It Reads

- SQLite database: repository root `face_records.db`
- Attendance log file: repository root `attendance_logs.txt`
- Status rule source reference: repository root `main_recognition.py`

## Features

- Dashboard summary cards for real registered employees and today's attendance
- Live monitoring panel based on the latest real recognition event
- Attendance history table with search and filters
- Registered employees table using real names from the SQLite database
- Admin login protected by `.env` credentials and signed local session cookies
- Delete employees directly from the dashboard with confirmation and backend cleanup
- Activity log viewer using the real log file
- Employee detail modal with attendance, late, and overtime history
- Status rules page mirroring the current Python time thresholds

## Required Local Environment Variables

Create a repository root `.env` file before starting the dashboard:

```text
ADMIN_USERNAME=admin
ADMIN_PASSWORD=your-password
SESSION_SECRET=your-long-random-secret
```

You can copy the included `.env.example` file and replace the values.

## Run Locally

This dashboard does not require an extra web framework or frontend package. It uses Python standard library modules such as `http.server` and `sqlite3`.

Required check:

```bash
python --version
```

If Python is not installed, Python 3.10+ is enough. There are no extra required packages just for the dashboard itself.

From the repository root:

```bash
cd web-dashboard
python run_dashboard.py
```

Then open:

```text
http://127.0.0.1:8000
```

You will be redirected to a login page first. After signing in, the dashboard session is required for all dashboard pages and API requests.

## Notes

- The dashboard is designed for local admin use.
- The original Python recognition app should continue running separately as the source of truth.
- The dashboard refreshes every 5 seconds to pick up new database or log changes.
- No fake employee names are generated by this web app. It only shows what exists in the SQLite database and log file.
- Runtime log files are stored in `web-dashboard/runtime_logs/`.
- Deleting an employee from the dashboard removes the person from `users`, `employees`, and structured `attendance_logs`. If `photo_path` exists and points inside the project, that file is also removed.
