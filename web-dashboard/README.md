# Local Attendance Admin Dashboard

This web app is a separate read-only layer for the existing Python face recognition project. It does not modify the original recognition scripts, database schema, or attendance rules.

## What It Reads

- SQLite database: `Face-Recognation-Based-Tracking-System-main/face_records.db`
- Attendance log file: `Face-Recognation-Based-Tracking-System-main/attendance_logs.txt`
- Status rule source reference: `Face-Recognation-Based-Tracking-System-main/main_recognition.py`

## Features

- Dashboard summary cards for real registered employees and today’s attendance
- Live monitoring panel based on the latest real recognition event
- Attendance history table with search and filters
- Registered employees table using real names from the SQLite database
- Activity log viewer using the real log file
- Employee detail modal with attendance, late, and overtime history
- Status rules page mirroring the current Python time thresholds

## Run Locally

From the repository root:

```bash
cd web-dashboard
python run_dashboard.py
```

Then open:

```text
http://127.0.0.1:8000
```

## Notes

- The dashboard is designed for local admin use.
- The original Python recognition app should continue running separately as the source of truth.
- The dashboard refreshes every 5 seconds to pick up new database or log changes.
- No fake employee names are generated by this web app. It only shows what exists in the SQLite database and log file.
